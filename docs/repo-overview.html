Backend — Python / FastAPI
🚀
main.py
backend/main.py
FastAPI app entry point. Owns all API routes, APScheduler jobs (daily cards, weekly digest), admin endpoints, and invite-code registration.
API routes Scheduler Admin endpoints Invite codes
🤖
agent_runner.py
backend/services/agent_runner.py
Generates each user's daily card. Assembles the full prompt (odds + injuries + memory + preferences) and calls the Claude API. Returns structured plays, leans, and quick reads.
Claude API call Card generation Prompt assembly
📊
grader.py
backend/services/grader.py
Auto-grades pending bets using ESPN box scores. After grading, calls memory.refresh_memory() to update stats. Non-prop bets (spreads, MLs, totals) require manual review.
Auto-grading ESPN scores W/L results
🧠
memory.py
backend/learning/memory.py
Computes 90-day rolling performance stats by market, sport, confidence tier, and odds bucket. Stores to agent_memory table. Formats stats into the daily prompt so the agent self-adjusts.
Performance stats ROI by market Prompt context Learning loop
📈
system_prompt.py
backend/esm/system_prompt.py
The full ESM framework (~3000 tokens). Defines agent identity as a professional gambler, Kelly sizing rules, edge thresholds, 0-play day logic, and EV-first philosophy. Prompt-cached.
Agent identity Kelly sizing Edge thresholds Prompt cache
🎲
odds_client.py
backend/esm/odds_client.py
Wraps The Odds API. Fetches live lines, spreads, moneylines, totals, and player props for active sports. Feeds raw odds data into the agent's daily context.
The Odds API Live lines Player props
🏟️
stats_client.py
backend/esm/stats_client.py
Wraps the ESPN public API (no key needed). Fetches injury reports, team news, scoreboard, and game context. Used by both the agent prompt and grader.
ESPN API Injury reports Scoreboard
⚙️
config.py
backend/esm/config.py
Single source of truth for active sports, enabled prop markets, and API keys. Edit here to turn sports on/off or add new market types.
Active sports Prop markets API keys
🔐
auth.py
backend/auth.py
JWT validation via Supabase. Exposes get_current_user() and require_admin() FastAPI dependencies used to protect routes.
JWT auth Admin check Supabase auth
🗄️
database.py
backend/database.py
Supabase client singleton. Import get_supabase() from here anywhere a DB connection is needed. Initialized once at startup with SUPABASE_URL + SUPABASE_SERVICE_KEY.
Supabase client Singleton
Frontend — Next.js 14
🖥️
frontend/app/
frontend/app/
Next.js App Router pages: dashboard (today's card), login, history (past cards), and preferences (user settings). Styled with Tailwind CSS and lucide-react icons.
Dashboard Login History Preferences
🔌
api.ts
frontend/lib/api.ts
All fetch calls from the frontend to the FastAPI backend live here. Single place to update base URLs, auth headers, and response types.
API calls Auth headers TypeScript
Database — Supabase (PostgreSQL + RLS)
profiles
Extends auth.users. Stores email, full_name, is_admin, is_active flags.
preferences
Per-user settings: sports, bet_types, risk_level, max_plays, unit_size.
cards
Daily AI-generated cards: plays, leans, quick_reads, pass_notes, raw JSON.
bets
Individual wagers extracted from cards. result starts as "pending", updated by grader.
invite_codes
Invite-only registration codes. Required to create a new account.
agent_memory ⚠️
90-day rolling stats JSON per user. Table not yet created — run migration in Supabase SQL Editor to activate the learning loop.

Scheduled Jobs (APScheduler)
Job Schedule What it does
run_daily_cards 9:30 AM MT daily Grades yesterday's bets → generates today's card for all active users → seeds memory stats
run_weekly_digest Monday 8:00 AM MT Logs the 7-day win/loss record for all active users

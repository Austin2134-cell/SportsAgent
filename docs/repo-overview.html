Backend Modules — Python / FastAPI
🚀
main.py
backend/main.py
FastAPI app entry point. Defines all API routes, wires up APScheduler jobs, handles invite-code registration, and exposes admin endpoints.
API routes Scheduler Admin Invite codes
🤖
agent_runner.py
backend/services/agent_runner.py
Builds each user's daily prompt from odds + injuries + memory + preferences, calls the Claude API, and saves the structured card (plays, leans, quick reads) to Supabase.
Claude API Card generation Prompt assembly
📊
grader.py
backend/services/grader.py
Auto-grades pending bets against ESPN box scores and writes W/L results to the bets table. Then triggers memory.refresh_memory() to update rolling stats.
Auto-grading ESPN scores W/L results
🧠
memory.py
backend/learning/memory.py
Computes 90-day rolling stats by market, sport, confidence tier, and odds bucket. Writes to agent_memory table. Formats stats into next-day prompt so the agent self-adjusts sizing.
Rolling stats ROI by market Self-adjustment Learning loop
📝
system_prompt.py
backend/esm/system_prompt.py
The full ESM framework (~3,000 tokens, prompt-cached). Defines the agent as a professional gambler, sets Kelly sizing rules, edge thresholds, 0-play-day logic, and EV-first philosophy.
Agent identity Kelly sizing EV-first Prompt cache
🎲
odds_client.py
backend/esm/odds_client.py
Wraps The Odds API. Fetches live spreads, moneylines, totals, and player props for all active sports. Feeds raw odds into the agent's daily context window.
The Odds API Live lines Player props
🏟️
stats_client.py
backend/esm/stats_client.py
Wraps the ESPN public API (no key needed). Fetches injury reports, team news, and scoreboard data. Used by both agent_runner (context) and grader (results).
ESPN API Injury reports Scoreboard
⚙️
config.py
backend/esm/config.py
Single source of truth for active sports, enabled prop markets, and API keys. Edit here to toggle sports on/off or add new market types without touching any other file.
Active sports Prop markets API keys
🔐
auth.py
backend/auth.py
JWT validation via Supabase. Exposes get_current_user() and require_admin() as FastAPI dependencies to protect any route that needs them.
JWT auth Admin guard Supabase auth
🗄️
database.py
backend/database.py
Supabase client singleton. Every module that needs a DB connection imports get_supabase() from here — initialized once at startup with SUPABASE_URL + SERVICE_KEY.
Supabase client Singleton
Frontend — Next.js 14 + Tailwind
🖥️
frontend/app/
frontend/app/
Next.js App Router pages: dashboard (today's card), login, history (past cards + record), and preferences (sports, unit size, risk level). Styled with Tailwind + lucide-react.
Dashboard Login History Preferences
🔌
api.ts
frontend/lib/api.ts
All fetch calls from the browser to the FastAPI backend live here. Single file to update base URLs, auth headers, and TypeScript response types.
API calls Auth headers TypeScript
Database — Supabase (PostgreSQL + RLS)
profiles
Extends auth.users. Stores email, full_name, is_admin, is_active flags per user.
preferences
Per-user betting settings: sports, bet_types, risk_level, max_plays, unit_size.
cards
Daily AI-generated cards: plays, leans, quick_reads, pass_notes, and raw JSON output.
bets
Individual wagers from each card. result starts as "pending", updated to W/L by grader.py.
invite_codes
Invite-only registration. A valid code is required to create a new account.
agent_memory ⚠️
90-day rolling stats JSON per user. Table not yet created in production — run the SQL migration in Supabase to activate the learning loop.
Scheduled Jobs
📅
run_daily_cards
Every day · 9:30 AM MT
Grades yesterday's bets via ESPN → refreshes agent memory stats → generates today's card for every active user via Claude.
📋
run_weekly_digest
Mondays · 8:00 AM MT
Logs the 7-day win/loss record and unit P&L for all active users. Used to track week-over-week profitability trends.
